# 安全政策

## 支持范围

安全更新以本仓库默认分支上的最新版本为准。历史版本可能不会单独提供安全补丁。

## 报告安全漏洞

请通过本仓库 GitHub “Security”页面的“Report a vulnerability”功能私密报告安全漏洞。请勿在公开 Issue、Discussion 或 Pull Request 中披露可利用细节、密钥、生产环境地址或个人数据。

报告中建议包含受影响版本、复现步骤、影响说明以及可行的修复建议。维护者将在确认报告后评估影响并协调修复与披露时间。

示例配置和默认地址仅用于本地开发。生产部署必须更换全部密钥、限制跨域来源、启用 TLS，并按照最小权限原则配置账号与服务权限。
